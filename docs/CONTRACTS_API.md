## How to Read Soroban Contract Docs

Soroban contract documentation follows Rust conventions with additional Soroban-specific patterns:

### Public Functions (`#[contractimpl]`)

All public contract entry points are defined inside `#[contractimpl]` blocks. These are the
functions you can call from off-chain code.

Example from Campaign contract:

```rust
/// Register a participant.
///
/// `leaf`  – the 32-byte leaf value committed in the Merkle tree for this
///           participant.  Must be `sha256(address_xdr_bytes)` for the
///           caller's address, computed by off-chain tooling.
///
/// `proof` – ordered list of sibling hashes for the Merkle path from
///           `leaf` to the stored root.  Pass an empty `Vec` when no root
///           is configured.
///
/// Returns `true` on first registration, `false` if already registered.
pub fn register(
    env: Env,
    participant: Address,
    leaf: BytesN<32>,
    proof: Vec<BytesN<32>>,
) -> Result<bool, Error>
```

**Key things to look for:**

- **`env: Env`** — The Soroban environment context (always the first parameter)
- **`Address`** — A Stellar account or contract address
- **`Result<T, Error>`** — Functions return `Ok(T)` on success or `Err(Error::Variant)` on failure
- **`pub fn`** — Public entry points callable from off-chain

### Error Types (`#[contracterror]`)

Each contract defines an `Error` enum with numeric error codes:

```rust
#[contracterror]
#[repr(u32)]
pub enum Error {
    Unauthorized = 100,
    OutsideTimeWindow = 101,
    // ...
}
```

Error codes are unique within each contract. See [ERROR_CODES.md](ERROR_CODES.md) for the full list.

### Events

Events are documented in the module-level doc comment at the top of each `lib.rs` file. Each event
has:

- **Topics** — Indexed event identifiers (used for filtering)
- **Data** — The event payload

Example from Rewards contract:

```
// Events:
// - `credit`: topics `(credit, user)`, data `amount: u64`
// - `claim`: topics `(claim, user)`, data `amount: u64`
```

### Storage Layout

Key-value storage keys are documented as constants in the source code:

```rust
const ADMIN: Symbol = symbol_short!("admin");
const BALANCE: Symbol = symbol_short!("balance");
const PARTICIPANT: Symbol = symbol_short!("partic");
```

Storage uses two tiers:

- **Instance storage** — Shared across all users (admin, config, total counts)
- **Persistent storage** — Per-user data (participant records)

## ABI JSON (Contract Interface)

The contract ABI (Application Binary Interface) is a JSON file generated by the Stellar CLI. It
describes all public functions, their parameters, and return types in a machine-readable format.

### Generating ABI JSON

```bash
# Generate ABI for the campaign contract
stellar contract build --manifest-path contracts/campaign/Cargo.toml
# Output: target/wasm32-unknown-unknown/release/trivela_campaign_contract.wasm

# Extract ABI from WASM
stellar contract inspect --wasm target/wasm32-unknown-unknown/release/trivela_campaign_contract.wasm
```

### Where to Find ABI Files

After building, ABI information can be extracted from the compiled WASM files:

```
target/wasm32-unknown-unknown/release/*.wasm
```

The Stellar CLI embeds contract spec metadata directly in the WASM binary during compilation. Use
`stellar contract inspect` to view the JSON representation.

### Using ABI JSON

ABI JSON is used by:

- **Frontend integrations** — Generate SDKs and type-safe clients
- **Indexers** — Parse contract events and operations
- **Testing tools** — Generate test harnesses automatically

## TypeScript Bindings

Type-safe TypeScript bindings for the contracts are auto-generated from the compiled WASM artifacts
(see [issue #328](https://github.com/FinesseStudioLab/Trivela/issues/328)).

### Regenerating Bindings

```bash
npm run contracts:build-bindings
```

### Location

Generated bindings are checked into the repository under:

```
frontend/src/contracts/
```

### Usage Example

```typescript
import { CampaignContract } from '../contracts/campaign.js';

const contract = new CampaignContract({
  contractId: 'C...',
  rpcUrl: 'https://soroban-testnet.stellar.org',
  networkPassphrase: 'Test SDF Network ; September 2015',
});

const isActive = await contract.is_active();
const participantCount = await contract.get_participant_count();
```
